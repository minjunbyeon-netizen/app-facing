## 1. TL;DR

CrossFit 글로벌 시장(14,000+ 박스, 추정 500만 엘리트 athlete)에 WOD 시작 전 30초 안에 페이싱 전략을 제공하는 앱은 현재 존재하지 않는다.

**North Star**: 주간 WOD 페이싱 계산 횟수. DAU가 아닌 실제 가치 사용(페이싱 계산 1회 = athlete가 WOD 전략을 데이터 기반으로 수립한 1회).

**5년 비전**: 한국 CrossFit 박스의 표준 운영 도구로 자리잡은 뒤, 영문 카피를 그대로 들고 글로벌 박스 네트워크로 진출한다. CrossFit LLC 공식 데이터 파트너십이 가시권에 들어온다.

**미션**: 엘리트 athlete가 WOD 시작 전 30초 안에 페이싱 전략을 받고, 박스 코치와 통합된 환경에서 다크 패턴 없이 동기를 유지하도록 한다.

**한 줄 브랜드 보이스**: "Games-Player의 언어로, 숫자로만 말한다."

## 2. Problem

### 2-1. 시장 규모

- 글로벌 CrossFit 박스: 14,000+ (CrossFit LLC 추정, 2024 기준)
- 글로벌 CrossFit 등록 athlete: 500만+ (추정, 비공개 데이터)
- 한국 박스: 약 600개 이상 (추정)
- 핵심 타깃: RX 이상 (Scaled 제외) athlete — 전체의 40~60%로 추정

### 2-2. 현재 도구 지형

| 도구 | 강점 | 결핍 |
|---|---|---|
| WODsmith | WOD 기록 | 페이싱 계산 없음. 박스 통합 없음 |
| SugarWOD | WOD 공유·리더보드 | 페이싱 없음. 코치 도구 제한적 |
| MyFitnessPal / Strong | 일반 피트니스 | CrossFit 전문성 전무 |
| Excel / 종이 | 코치 자체 제작 | 멤버 전달 불가. 자동화 없음 |
| 카카오톡 | 코치-멤버 메시지 | 구조화 없음. WOD·숙제 분리 없음 |

결론: **페이싱 + 박스 통합 + 화이트햇 동기부여를 하나의 앱에서 제공하는 솔루션이 없다.**

### 2-3. 실제 Pain

코치 관점: WOD를 카카오톡으로 보내고, 숙제는 메모장으로 보내고, 출석은 엑셀로 관리한다. 3개 채널을 동시에 운영하는 인지 비용이 높다.

athlete 관점: WOD 공지 받고 박스 입장 전까지 "어떻게 분할할지"를 스스로 추산해야 한다. 1RM 기반 정량 계산 도구가 없다. 앱이 있어도 "할 수 있어요!" 류의 공허한 동기부여 카피가 엘리트 사용자 신뢰를 깎는다.

## 3. Tenets

FACING 운영 원칙. 우선순위 순.

**T1. Athlete-First, 다크 패턴 거부.**
랜덤 보상 · 회복 결제 · Lv 차감 · FOMO 루프 전면 차단. 단기 retention 지표를 이유로 이 원칙을 타협하지 않는다. 엘리트 athlete는 속이는 앱을 신뢰하지 않는다.

**T2. 결정론적 보상만.**
unlock 임계는 공개된 조건(Engine 80+ 5회 → IRON LUNG)에 근거한다. 조건 불투명 배지 · 확률 박스 없음. 보상 예측 가능 = athlete 자율성 보장.

**T3. 데이터는 박스 단위로 분리.**
개인 Engine 점수와 박스 리더보드는 별도 저장·별도 공개 경로. 코치가 멤버 개인 데이터를 임의로 열람하거나 강제 공개할 수 없다. 개인 데이터 소유권은 사용자에게 있다.

**T4. 영문 카피 고정, 글로벌 번역 비용 0.**
동작명 · 등급명 · 메트릭은 CrossFit 국제 표준 영문 그대로. "Thruster를 스러스터로 번역"하지 않는다. 글로벌 진출 시 추가 번역 없이 동일 UX 제공. 한국어 보조 캡션만 제거하면 된다.

**T5. 박스 코치 도구 의존성이 B2B 해자.**
코치가 FACING으로 WOD를 등록하고, 숙제를 보내고, 멤버를 관리하는 순간 — 멤버도 FACING을 써야 한다. 양방향 lock-in(코치 ↔ 멤버)이 경쟁 진입 장벽이 된다.

**T6. 페이싱 계산은 백엔드 독점.**
알고리즘은 `services/facing/engine/`에만. 앱은 UI만 담당. 계산 로직 앱 내 구현 금지. 지적재산권 보호 + 알고리즘 개선이 즉시 모든 사용자에 반영됨.

## 4. Approach

### 4-1. 5 핵심 가치 통합

| 가치 | 설명 | 사용자 |
|---|---|---|
| 페이싱 (Pacing) | WOD 시작 전 Split + Burst + 예상 완주 시간 | 전 사용자 |
| WOD 공지 | 코치 게시 → 멤버 수신 + 리더보드 | 박스 코치·멤버 |
| 쪽지 (Inbox) | 코치-멤버 1:1·그룹·전체 메시지 | 박스 코치·멤버 |
| 숙제 (Assignment) | 구조화 과제(동작·세트·하중·마감) + 수행 기록 | 박스 코치·멤버 |
| 업적 (Gamification) | Tier·칭호·시즌 배지·Streak·Level — 결정론적 | 전 사용자 |

### 4-2. 2 레이어 funnel (v1.1 ModeSelect 정합)

혼자 사용자(Solo 모드, `app_user`) → 페이싱 + Engine 측정으로 가치 경험 → **Settings 또는 Profile 박스 검색 진입점에서 Member 모드로 전환** → 박스 가입 신청 → 코치 WOD + 숙제 + 쪽지로 retention 극대화 → 코치가 FACING으로 전환 → 박스 단위 SaaS 구독 전환.

**Solo→Member 전환율 (월 ≥ 10%) = lock-in 루프의 첫 dial** (§8 정합). 미달 시 박스 검색 진입점 UX 또는 박스 공급 (파트너 박스 수) 부족 신호.

B2C: 개인 athlete (Solo 모드). 무료 진입. 페이싱·Engine·업적 경험.
B2B: 박스 owner (Coach 모드). WOD 관리·멤버 관리·코치 도구. SaaS 수익원(Y2~).
중간 layer: 박스 가입 멤버 (Member 모드). 양면 시장의 양쪽을 동시에 끌어당기는 핵심 사용자.

### 4-3. 글로벌 확장 설계

영문 전문 용어 유지(V6·V8·V11)로 로컬라이제이션 비용이 사실상 없다. 한글 캡션만 제거하면 영문 UX 완성. CrossFit은 전 세계에서 동일한 WOD·동작명을 사용하므로 앱 콘텐츠가 즉시 통용된다.

## 5. Metrics

### 5-1. North Star Metric (NSM)

**주간 WOD 페이싱 계산 횟수 (Weekly Pacing Calculations)**

선택 근거:
- DAU/MAU는 허수(앱 열기만 해도 증가)
- 페이싱 계산 1회 = athlete가 WOD 전에 데이터 기반 전략을 수립했다는 증거
- 이 지표가 늘면 박스 내 FACING 사용 문화가 형성됨(코치·동료 효과)
- 매출·retention과 직접 상관(계산 → WOD 세션 완료 → 기록 → 재방문)

NSM 보조 지표 2개:
- RX+ 이상 사용자 비율 (엘리트 타깃 농도 유지)
- 박스 코치 주간 활성률 (B2B retention 건강도)

### 5-2. 5년 KPI (모두 추정치 — 검증 필요)

| 연도 | 사용자 | 박스 계약 | NSM (주간 계산) | 매출 |
|---|---|---|---|---|
| Y1 (2026) | 1만 명 | 50개 | 5,000회/주 | 0 (무료 MVP) |
| Y2 (2027) | 5만 명 | 200개 | 3만 회/주 | 박스 SaaS 시작 · 추정 월 2,000만 원 |
| Y3 (2028) | 15만 명 | 500개 · 글로벌 진출 시작 | 10만 회/주 | 추정 월 8,000만 원 |
| Y4 (2029) | 40만 명 | 1,500개 · 해외 박스 포함 | 30만 회/주 | 추정 월 3억 원 |
| Y5 (2030) | 100만 명 | 3,000개 | 80만 회/주 | 추정 월 8억 원 · CrossFit 파트너십 협상 |

수익 모델 (추정):
- B2C 개인: 무료 기본 / 프리미엄 구독(고급 페이싱 분석 · 히스토리 무제한) 추정 월 9,900원
- B2B 박스: SaaS 구독(WOD 관리·코치 도구·멤버 분석) 추정 월 99,000~299,000원/박스
- Y3 글로벌: 달러 과금($9.9/박스/월)

### 5-3. Retention 목표 (추정)

- D30 retention ≥ 40% (페이싱 사용자 기준)
- 박스 멤버 M3 retention ≥ 70% (코치 도구 lock-in 효과)
- 코치 M6 retention ≥ 80% (전환 비용 높음)

## 6. Risks / Trade-offs

### 6-1. 시장 진입 위험

| 위험 | 확률 | 완화 |
|---|---|---|
| 엘리트 타깃 = 니치 시장 → 사용자 수 제한 | 높음 | NSM은 횟수(사용 강도). 1만 명 고강도 사용자가 100만 명 라이트 사용자보다 가치. 박스 SaaS 수익은 사용자 수와 독립. |
| CrossFit 박스 디지털화 속도 느림 | 중간 | 혼자 사용자(Solo 모드) 먼저 mass. 박스 가입은 2단계 funnel. |
| **Solo 영구화 → lock-in 미작동 (v1.1)** | **중간** | **WOD 탭·Profile 박스 검색 진입점 강조 + Solo→Member 월 ≥ 10% KPI 추적. 미달 시 박스 검색 UX·박스 공급 점검 (§8 R6 정합)** |
| WODsmith · SugarWOD 기능 복제 | 낮음 단기 / 높음 중기 | 페이싱 알고리즘은 백엔드 독점 IP. 박스 lock-in이 전환 비용. |
| CrossFit LLC 직접 앱 출시 | 낮음 | 파트너십 전환 가능성이 더 높음. B2B 데이터 파트너 포지셔닝. |

### 6-2. 다크 패턴 거부 = 단기 지표 약화

- Duolingo·Strava 모델(Streak 압박·FOMO)은 단기 DAU를 높인다.
- FACING은 이를 거부한다. 단기 DAU는 낮을 수 있다.
- 트레이드오프: 엘리트 athlete 신뢰 확보 vs 단기 지표. 장기 retention과 유료 전환율이 보상한다.
- 리스크: VC는 초기 DAU를 본다. 이 트레이드오프를 투자자에게 명확히 설명해야 한다.

### 6-3. 알고리즘 의존성

- 페이싱 알고리즘 정확도가 앱 가치의 핵심이다.
- 잘못된 분할 전략 제안 → athlete 부상 또는 WOD 실패 → 신뢰 붕괴.
- 백엔드 엔진의 지속적 개선과 논문 기반 업데이트가 필수다.
- 논문·공식 자료 SSOT: `services/facing/docs/refer/{카테고리}/findings.md`.

### 6-4. 박스 코치 온보딩 마찰

- 코치가 FACING으로 전환하려면 기존 카카오톡·엑셀 워크플로를 바꿔야 한다.
- 온보딩 마찰 최소화 = MVP의 가장 큰 실행 과제.
- 해결 방향: 코치가 첫 WOD를 3분 안에 등록할 수 있어야 한다. 기존 채널 병행 허용.

## 8. FAQ

**Q1. 왜 엘리트 타깃인가? 일반 피트니스 시장이 더 크지 않나?**

일반 피트니스 앱 시장은 이미 MyFitnessPal · Nike Training Club · Strava가 지배한다. 후발주자가 일반 시장으로 진입하면 마케팅 비용이 기하급수로 증가한다. 엘리트 CrossFit athlete는 니치지만 지출 의향이 높고 커뮤니티 영향력이 강하다. 한 박스의 RX+ athlete가 "이거 써봐"라고 하면 박스 전체가 따른다. 바이럴 구조가 내재되어 있다. 또한 엘리트 타깃이 쓰는 앱이 Scaled 사용자를 끌어당긴다 — 반대는 성립하지 않는다.

**Q2. 시장 사이즈가 너무 작지 않나?**

글로벌 CrossFit 등록 athlete 500만 명 추정. 한국만 해도 활성 박스 600개, 멤버 수 10만 명 이상으로 추정. B2C 사용자 수보다 B2B 박스 SaaS 단가가 더 중요하다. 박스 3,000개 × 월 $150 = 월 $450,000 ARR. 소규모 VC 투자 회수 가능 규모. CrossFit 공식 파트너십이 달성되면 시장이 재정의된다.

**Q3. 글로벌 진출 시 언어 장벽은?**

FACING의 핵심 UX(동작명·등급·메트릭)는 이미 영문 단독이다. 한글 캡션만 영문으로 전환하면 된다. 디자인 시스템과 카피 규칙(V1~V11)이 이미 글로벌 대응 전제로 설계되어 있다. 번역 비용 사실상 0.

**Q4. 페이싱 알고리즘이 틀리면 어떻게 되나?**

잘못된 분할 전략 → athlete 오버페이스 → WOD 실패 또는 부상 위험. 이것이 가장 큰 단일 리스크다. 완화 방법: (1) 논문 기반 알고리즘 업데이트 지속 (`services/facing/docs/refer/`), (2) 결과 화면에 근거 표시 (athlete 스스로 검증 가능), (3) "추천이지 명령이 아님" 포지셔닝. 알고리즘 오류는 단기 신뢰 손상이지만, 투명한 근거 표시가 완충재다.

**Q5. SugarWOD가 페이싱 기능을 복제하면?**

SugarWOD는 WOD 기록 중심 플랫폼이다. 페이싱 알고리즘은 6 카테고리(Power · Olympic · Gymnastics · Cardio · Metcon · Body) 가중 Engine 점수 기반 개인화 계산이다. 단순 복제는 어렵다. 더 중요한 것은 박스 lock-in이다. 코치가 FACING을 쓰는 순간 멤버도 전환해야 한다. SugarWOD가 기능을 복제해도 전환 비용이 높다.

**Q6. 왜 다크 패턴을 거부하는가? 매출에 손해 아닌가?**

단기적으로는 손해다. Duolingo 식 Streak 압박은 단기 DAU를 높인다. 그러나 CrossFit 엘리트 athlete는 이미 강한 내재적 동기가 있다. 외재 압박을 혐오한다. "100일 잃습니다" 카피를 보는 순간 앱을 삭제한다. 다크 패턴 거부 = 엘리트 신뢰 = 장기 retention = 유료 전환. 이 경로가 더 수익성이 높다고 판단한다.

**Q7. CrossFit LLC 파트너십이 왜 Y5인가? 더 빨리 접근할 수 없나?**

CrossFit LLC는 공식 앱 생태계에 보수적이다. 데이터 볼륨(박스 수·athlete 수·페이싱 계산 횟수)이 협상력을 만든다. Y1~Y3에 충분한 데이터 축적 없이 접근하면 의존적 파트너십이 된다. Y5 접근 = 대등한 협상.

**Q8. Whoop · Garmin 연동은 왜 Y4인가?**

MVP 단계에서 바이오메트릭 연동은 알고리즘 복잡성을 급증시킨다. HRV · 수면 · 회복도 데이터를 페이싱에 반영하려면 백엔드 엔진 재설계가 필요하다. 먼저 1RM · UB · 카디오 페이스 기반 핵심 페이싱을 검증한다. 확장은 검증 이후.

**Q9. iOS는 언제 출시하나?**

Y2 Q1 목표. Android MVP로 한국 초기 사용자 확보 → iOS로 글로벌 진출 준비. Android 우선 전략은 한국 시장 점유율(Android 70%+)에 최적화된 판단이다.

**Q10. 창업자 1인이 5년 로드맵을 실행할 수 있나?**

MVP까지는 가능하다. Y2 박스 SaaS 이후는 코치 온보딩 · CS · 글로벌 영업 인력이 필요하다. Y1에 첫 수익이 발생하면 채용을 시작한다. 기술 스택(Flutter + Flask)은 소규모 팀으로 유지 가능한 수준으로 설계되어 있다.
