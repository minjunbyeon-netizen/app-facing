## Sprint 로드맵 권장

### Sprint 7a (즉시 — P0 이탈 차단)
1. 용어 툴팁 (`1RM` · `AMRAP` · `Unbroken` 등 최초 등장 시 1줄 팝오버) — Onboarding · Builder
2. Rationale 최소 구현 ("이 점수는 6지표 평균 + 지표별 기여도 %") — Result 화면
3. WOD 빌더 검색 + 단위 토글 고정 — Calc Custom
4. Masters 연령 입력 필드 (생년월일 → 35+/45+/55+ 자동 분류) — Onboarding Step 1
5. Signup 약관·개인정보 placeholder 링크 — Signup (이미 이전 Persona v1.16 Auth SSOT 권장)

### Sprint 7b (1주 내 — P1 유지율)
1. History 추이 그래프 (Engine 시간축 라인차트)
2. Engine 약점 카테고리 자동 강조 + AI 코멘트
3. 백분위 ("상위 N%") 서버 연동
4. 프라이버시 정책 화면 + 탈퇴 버튼
5. 기능 발견성 첫 실행 툴팁 (BottomNav 5탭)

### Sprint 8+ (장기 — P2 리텐션·바이럴)

## 메타 통찰 (3줄)

1. **G · A · S 카테고리가 이탈 3대 근원**: 계산 근거 불투명(G) · 첫 진입 장벽(A) · 빌더 마찰(S) 동시에 해결 안 되면 Scaled→RX 전환기 페르소나 대부분이 2회차 전 이탈.
2. **Masters 세그먼트(O·P)는 완전한 사각지대**: 35+ 연령층은 CrossFit 활동 인구 35%+ 비중인데 현재 앱에 전혀 대응 없음. v1.16 자간 복원은 시작일 뿐, Masters 전용 기준·접근성 일괄 개편 필요.
3. **공유(Q)·코치(R) 카테고리는 P2이지만 바이럴·B2B 레버리지**: 결과 카드 이미지 하나만 구현해도 신규 유입 경로 생기고, 코치 연동은 박스 단위 단체 도입으로 연결.
